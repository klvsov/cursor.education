Основні команди в GIT

git config --global user.name blabla
Глобальне налаштування (для всіх локальних репозиторіїв) - ім'я користувача
git config --global user.email blabal@bla
Глобальне налаштування (для всіх локальних репозиторіїв) - електронна пошта
git init
Створити локальний репозиторій в активній папці
git status
Переглянути нові і змінені файли, які потрібно зафіксувати
git add
Додати до індексованих вказаний файл або всі для подальшої фіксації
git commit
Зафіксувати індексовані зміни і зберегти їх в історію версій
git log
Історія коммітів для поточної гілки (branch)
git checkout
Перейти до потрібної гілки
git branch
Список гілок з позначенням вибраної
git push
Завантажити всі зміни з вибраної гілки локального репозиторія у віддалений
git pull
Завантажити зміни з віддаленого репозиторію і об'єднати зміни з локальним
git merge
Злиття однієї або декількох гілок в поточну
git clone
Скопіювати віддалений репозиторій з історією змін
git revert
Скасувати комміт
git reset
Відмінити індексацію файлу зі збереженням його вмісту
git rebase
Перезаписати історію коммітів при злитті гілок
git cherry-pick
Перенесення окремих коммітів між гілками
git stash
Приховати зміни в файлах і зберегти їх окремо, в подальшому можна відновити

Файл .gitignore - це файл, в якому записуються окремі папки, файли або типи файлів, які Git не повинен індексувати. Це може бути папка з файлами препроцесора, системні файли Webstorm або PHPStorm і тому подібне
